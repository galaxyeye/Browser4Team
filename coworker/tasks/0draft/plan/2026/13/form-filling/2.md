# Create CLI to Fill a Form

Fill a complex form with multiple fields and submit it. The form should include various types of input fields, such
as text inputs, dropdowns, checkboxes, and radio buttons. After filling the form, take a snapshot of the filled form
before submitting it.

A sample workflow might look like this:

```shell
browser4-cli open https://example.com/form
browser4-cli fill --data-file=form-data.json
browser4-cli click e54
browser4-cli snapshot
browser4-cli close
```

In this example, `form-data.json` would contain the necessary data to fill the form, structured in a way that the CLI
can interpret and apply to the corresponding form fields. The `click e54` command simulates clicking the submit button
(assuming `e54` is the reference for the submit button), and the snapshot command captures the state of the form after
it has been filled but before submission.

Example `form-data.json` structure:

```json
{
  "e1": "John Doe",
  "e2": "Male",
  "e3": "23",
  "e4": [
    "Option1",
    "Option3"
  ],
  "e5": true
}
```

In this example, `e1` might correspond to a text input for the name, `e2` to a dropdown for gender, `e3` to a text input
for age, `e4` to checkboxes for multiple options, and `e5` to a checkbox for agreeing to terms and conditions.

# Create a CLI test to fill a form

Fill a complex form with multiple fields and submit it. The form should include various types of input fields, such as text inputs, dropdowns, checkboxes, and radio buttons. After filling the form, take a snapshot of the filled form before submitting it.

This is a test to ensure that the CLI can handle filling out forms correctly, including interacting with different types of form elements and capturing the state of the form before submission.

A sample workflow might look like this:

```shell
browser4-cli open https://example.com/form
browser4-cli fill e40 "John Doe"
browser4-cli fill e41 "Male"
browser4-cli fill e42 "23"
browser4-cli check e43
browser4-cli select e44 "Option1"
browser4-cli snapshot
browser4-cli close
```

In this example, `e40` might correspond to a text input for the name, `e41` to a dropdown for gender, `e42` to a text input for age, and `e54` to the submit button. The snapshot command captures the state of the form after it has been filled but before submission, allowing you to verify that the form was filled out correctly.

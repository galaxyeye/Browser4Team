### Batch Execution Testing

Execute multiple commands in a single invocation. Commands can be passed as
quoted arguments or piped as JSON via stdin. This avoids per-command process
startup overhead when running multistep workflows.

In the previous step, we have implemented the batch execution feature in the backend. Now we will test it to ensure it works as expected.

Create a comprehensive test suite that covers various scenarios of batch execution, including:

- Valid command sequences
- Invalid commands
- Error handling with and without the --bail option
- Performance benefits compared to individual command execution
- Edge cases, such as empty command lists or commands with special characters
- Integration tests that simulate real-world workflows using batch execution
- A complex webpage with multiple elements and interactions to test the robustness of batch execution
- A complex form submission workflow that involves multiple steps, such as filling out a form, submitting it, and verifying the results

```bash
# Argument mode: each quoted argument is a full command
browser4-cli batch "open https://example.com" "snapshot"

# With --bail to stop on first error
browser4-cli batch --bail "open https://example.com" "click e1" "screenshot"

# Stdin mode: pipe commands as JSON
echo '[
  ["open", "https://example.com"],
  ["snapshot"],
  ["click", "e1"],
  ["screenshot", "result.png"]
]' | browser4-cli batch --json
```

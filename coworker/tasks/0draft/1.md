# Fix backend service location to in CLI

The current logic for determining the backend service location in the CLI is as follows:

```rust
async fn resolve_server_launch_spec(port: u16) -> Result<ServerLaunchSpec, String> {
    if let Some(repo_root) = find_browser4_root() {
        return build_maven_launch_spec(&repo_root, port);
    }

    let jar_path = find_or_download_jar().await?;
    Ok(build_jar_launch_spec(&jar_path, port))
}
```

The expected behavior is:

1. If the CLI is run from within a Browser4 repository, it should automatically build and launch the server using the local source code.
2. If the CLI is run from outside a Browser4 repository, it should fall back to using the latest released `Browser4.jar` as the backend service.
3. In both cases, the CLI should ensure that the backend service is launched on the specified port (default 8182) and that the CLI can communicate with it successfully.

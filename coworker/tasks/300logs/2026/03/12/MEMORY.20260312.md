# MEMORY.20260312.md
- **Removed Browser4 MCP client feature while preserving MCP server support:** Deleted the agentic MCP client bootstrap/config/manager/plugin/executor stack, removed client-only examples/tests/mock servers/docs, and kept the Browser4 MCP server surfaces (`Browser4MCPServer`, MCP REST controller, launcher `--app=mcp`) intact. I also updated `AgentToolExecutor` so Browser4 now routes built-in tools and skills only, which cleanly removes Browser4 acting as an MCP client without affecting Browser4 acting as an MCP server. **Outcome:** Completed the requested product-scope change and aligned dependent docs/tests with the supported feature set.
- **Stabilized downstream server code/tests after the client removal:** Adjusted kept MCP server call sites and tests to use `ToolCallResult.evaluate` instead of assuming `executeToolCall()` returns raw `TcEvaluate`, including fixes in `Browser4MCPServer`, `MCPToolController`, server tests, and `MCPToolControllerTest`. **Outcome:** Validation passed with `./mvnw.cmd -pl pulsar-agentic,pulsar-rest,browser4\\browser4-agents -am -DskipTests compile`, `./mvnw.cmd -pl pulsar-agentic -am -Dtest=ToolCallSpecificationRendererTest,Browser4MCPServerTest,Browser4MCPServerToolManagerTest -Dsurefire.failIfNoSpecifiedTests=false test`, `./mvnw.cmd -pl pulsar-rest -am -Dtest=MCPToolControllerTest -Dsurefire.failIfNoSpecifiedTests=false test`, and `./mvnw.cmd -pl browser4\\browser4-agents -am -Dtest=Browser4LauncherTest -Dsurefire.failIfNoSpecifiedTests=false test`.
- **Lessons learned:** After changing shared agentic APIs, rerun the full affected Maven reactor from the top instead of resuming downstream modules against stale artifacts. Also, removing the Browser4 MCP client feature does not mean eliminating the MCP client SDK from test scope—kept MCP server protocol tests still need a client implementation to talk to the server.

# MEMORY.20260415.md
## Daily Memory - 2026-04-15

- Reworked `browser4-cli batch` from a recursive frontend-only wrapper into a real backend-executed flow: the Rust CLI now parses batch input, compiles each logical CLI command into backend batch steps, preserves local session/selector/mouse state around the response, and saves snapshot/screenshot artifacts after a single `command_batch` MCP call. On the server side, `pulsar-rest` now exposes `command_batch`, executes supported browser/session commands sequentially in one request, and returns structured per-step results so the CLI can keep existing batch UX such as `--json`, `--bail`, explicit snapshot output, and ordered failure reporting. Lesson: when batching an existing CLI surface for performance, keep the CLI as the syntax/state adapter but move execution to one backend request, and model multi-step commands like `open <url>` as one logical CLI command backed by multiple internal batch steps so behavior stays correct without reintroducing per-command HTTP overhead.
